# Implementation Plan: Sprint 4 — AI Commerce Intelligence

Este documento detalla el plan de ingeniería para construir el motor de inteligencia de comercio y orquestación de lanzamiento en **MIleGo Sprint 4**. Transformamos el cerebro de LIAM para generar y persistir el **Launch Blueprint** como fuente única de verdad, incorporando un gestor de costos de llamadas IA y un modo interactivo de copiloto comercial.

---

## 🛰️ El Paradigma del Launch Blueprint (Single Source of Truth)

```
                       [ RAW PRODUCT ]
                              │
                              ▼
                       [ LIAM BRAIN ]
                              │
                              ▼
                   [ LAUNCH BLUEPRINT ]
             (Persistido en PostgreSQL JSONB)
                              │
      ┌───────────────────────┼───────────────────────┐
      ▼                       ▼                       ▼
 [ STOREFRONT ]         [ META ADS ]           [ WA MESSAGES ]
  Lee el título,         Lee ganchos,           Lee respuestas
  slug y combos          copias y UGC           a objeciones
```

## 2. Modo Copiloto Comercial (Preguntas de LIAM)

Si un producto carece de información crítica durante el análisis en `LaunchEngine`, LIAM no generará un plan de lanzamiento automático. 
*   **Comportamiento:** Establecerá la clave `decision` en `"need_info"`, poblará el arreglo `missing_details` y bloqueará la publicación de la landing storefront hasta que el operador ingrese los datos faltantes.

## 3. Telemetría de Costo IA (AI Cost Manager)

Creamos la tabla `ai_requests_log` para registrar y auditar el consumo y coste monetario de cada consulta de LLM:

### Schema: `ai_requests_log`
*   `id`: serial primary key.
*   `provider`: string (e.g. `gemini`, `openai`).
*   `model`: string (e.g. `gemini-pro`, `gpt-4o`).
*   `prompt_tokens`: integer.
*   `completion_tokens`: integer.
*   `cost_usd`: decimal(10, 6).
*   `duration_ms`: integer.
*   `cache_hit`: boolean.
*   `prompt_template`: string (nombre del prompt, e.g. `seo.md`).
*   `created_at`: timestamp.
