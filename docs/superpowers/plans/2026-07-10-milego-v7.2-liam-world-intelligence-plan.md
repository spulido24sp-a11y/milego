# Implementation Plan: Sprint 5 — LIAM World Intelligence Engine

Este plan detalla el diseño de datos, la estructura modular y la suite de validación para construir el **World Intelligence Engine** (Motor de Inteligencia del Mundo) en **Sprint 5**. Esta actualización dota a LIAM de la capacidad de explorar de manera autónoma, relacionar información de éxito en un Knowledge Graph y orquestar experimentos comerciales A/B.

## 1. Diseña de Base de Datos (PostgreSQL Knex)

Para soportar la persistencia del Knowledge Graph, la caza de oportunidades y los experimentos, crearemos tres nuevas tablas relacionales en la base de datos:

### A. Tabla `knowledge_graph_nodes` (Entidades del Grafo)
*   `id`: serial primary key.
*   `type`: string (e.g. `'product'`, `'pain'`, `'hook'`, `'offer'`, `'audience'`).
*   `value`: text (el valor real, ej: `"Corrector de postura"`, `"Combo x2"` o `"Dolor lumbar"`).
*   `metadata`: jsonb (atributos adicionales).
*   `created_at`: timestamp.

### B. Tabla `knowledge_graph_edges` (Relaciones y Telemetría de Rendimiento)
*   `id`: serial primary key.
*   `source_node_id`: integer references `knowledge_graph_nodes.id`.
*   `target_node_id`: integer references `knowledge_graph_nodes.id`.
*   `relationship`: string (e.g. `'solves'`, `'converts_with'`, `'targets'`).
*   `times_used`: integer default 0.
*   `avg_ctr`: decimal(5, 2) default 0.00.
*   `avg_roas`: decimal(5, 2) default 0.00.
*   `memory_score`: integer default 50 (puntuación acumulada de 1-100).
*   `updated_at`: timestamp.

### C. Tabla `market_opportunities` (Productos Cazados)
*   `id`: serial primary key.
*   `source`: string (e.g. `'dropi'`, `'temu'`, `'trends'`).
*   `external_sku`: string.
*   `name`: string.
*   `estimated_margin`: decimal(5, 2).
*   `trend_status`: string (e.g. `'rising'`, `'stable'`).
*   `hunter_decision`: string (e.g. `'qualified'`, `'ignored'`).
*   `created_at`: timestamp.

### D. Tabla `ab_experiments` (Experimentos A/B de LIAM)
*   `id`: serial primary key.
*   `product_id`: integer references `products.id`.
*   `element_type`: string (e.g. `'offer'`, `'hook'`, `'landing'`).
*   `variation_a`: string.
*   `variation_b`: string.
*   `impressions_a`: integer default 0.
*   `impressions_b`: integer default 0.
*   `conversions_a`: integer default 0.
*   `conversions_b`: integer default 0.
*   `status`: string (e.g. `'active'`, `'completed'`).
*   `winner`: string.

## 2. Los 9 Sub-Motores de World Intelligence

Implementaremos los servicios bajo `backend/src/brain/world-intelligence/`:

1.  **WorldScanner (`scanner.js`):**
    *   Simula e integra web scraping o llamadas API a Google Trends, Dropi y competidores.
2.  **KnowledgeGraphService (`graph.js`):**
    *   Inserta, conecta y actualiza las relaciones entre nodos y edges en base de datos.
3.  **ConfidenceEngine (`confidence.js`):**
    *   Evalúa puntuaciones compuestas (Product, Offer, Competition). Si el promedio es < 70%, veta el lanzamiento y pide confirmación.
4.  **StrategyEngine (`strategy.js`):**
    *   Recomienda CPA objetivo, ROAS mínimo ideal, AOV esperado y bundles comerciales específicos basándose en datos del grafo.
5.  **OpportunityHunter (`hunter.js`):**
    *   Orquesta al Scanner para descubrir productos y califica cuáles merecen ser presentados en Mission Control.
6.  **ExperimentEngine (`experiments.js`):**
    *   Gestiona el registro de tests A/B y el cálculo de significancia estadística para coronar ganchos ganadores.
7.  **RiskEngine (`risk.js`):**
    *   Calcula el índice de riesgo logístico (peso, volumen) y saturación competitiva.
8.  **MemoryScoreManager (`memory.js`):**
    *   Evalúa la efectividad del conocimiento guardado (CTR y ROAS históricos).
9.  **LearningLoop (`learning-loop.js`):**
    *   Toma la data real de ventas del checkout de MIleGo y actualiza los pesos de las relaciones en el Knowledge Graph.
